# Third-Party Implementors Registry

> *Auditus praecedit responsum.* — Hearing precedes response.

This document tracks external researchers, authors, and practitioners who have
engaged with Generative Collapse Dynamics (GCD), the Universal Measurement
Contract Protocol (UMCP), or Retro-Coherent Field Theory (RCFT). Each entry
records **what was produced**, **where it lives**, and **the nature of the
engagement** (conceptual vs. computational).

## Classification System

Each implementor is classified by **engagement tier** and **engagement type**.

### Engagement Tier

| Tier | Criteria |
|:----:|----------|
| **A** | Co-authored publication with Paulus, or independent computational reproduction of Tier-1 identities |
| **B** | Institutional affiliation + published work engaging GCD/UMCP/RCFT concepts substantively |
| **C** | Published engagement (blog, preprint, paper) that references GCD/UMCP/RCFT without computational reproduction |

### Engagement Type

| Type | Meaning |
|------|---------|
| **Weld** | Computational engagement — ran the kernel, reproduced identities, validated against frozen contract |
| **Gestus** | Conceptual engagement — referenced, discussed, or applied GCD vocabulary without running the protocol |

## How to Add a New Implementor

Append a new entry under the appropriate tier heading using the template below.
Update the Statistics section. Run `python scripts/update_integrity.py` after
editing.

```markdown

| Field | Value |
|-------|-------|
| **Affiliation** | Institution or Independent |
| **ORCID** | (if known) |
| **Platform** | URL to profile or publication |
| **Engagement type** | Weld or Gestus |
| **Key papers** | Title (DOI if available) |
| **Verified** | Yes / Partial / No — method of verification |
| **Notes** | Nature of engagement, what was produced, any caveats |
```

## Promotion Criteria

An implementor may be promoted to a higher tier when:

- **C → B**: Publishes a substantive paper engaging GCD/UMCP concepts with
  institutional backing or peer review.
- **B → A**: Either (a) co-authors a publication with the GCD maintainer, or
  (b) independently reproduces Tier-1 kernel identities (F + ω = 1, IC ≤ F,
  IC = exp(κ)) computationally and publishes the reproduction with verifiable
  artifacts.
- **Gestus → Weld**: Demonstrates computational engagement — runs the kernel,
  reproduces frozen-contract identities, or validates a casepack against the
  protocol. Evidence must be verifiable (script output, notebook, Zenodo
  deposit, or equivalent).
